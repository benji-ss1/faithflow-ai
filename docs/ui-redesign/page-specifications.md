# Page Specifications

## Overview Dashboard

### Required blocks

- welcome card
- today’s service card
- next service checklist
- AI health card
- audio setup status
- projector setup status
- recent sermons
- imports waiting review
- storage and billing status
- team activity

### Behavior

- top row should prioritize today/next service
- setup/health cards should compress into clear status signals
- empty state should still feel useful, especially for new churches

## Organization

### Required content

- church profile
- location
- timezone
- congregation size
- denomination / style
- default Bible translation
- CCLI number
- branding / logo
- service schedule

### Behavior

- split into profile, worship defaults, and branding sections
- use editable cards, not one giant form wall

## Applications

### Required cards

- FaithFlow Presenter
- FaithFlow Archive
- FaithFlow AI
- future FaithFlow Livestream
- devices connected
- Bible licenses
- app status

### Behavior

- this page should feel like a product/module control center
- show enabled vs future products clearly

## Subscriptions

### Required content

- current plan
- seats
- trial state
- AI usage
- storage usage
- upgrade CTA
- cancel / manage

### Policy note

- billing failures must not hard-stop Sunday live operation
- UI should show grace-period / admin-warning state, not panic state

## Billing

### Required content

- payment method
- invoices / receipts
- VAT / tax exemption
- billing contact
- usage charges

### Behavior

- finance workflows should feel clean and trustworthy
- invoice history should be table-first with downloadable rows

## Bible Library

### Required content

- public-domain translations
- licensed translation slots
- lock states
- license-required badges
- manage Bible licenses

### Behavior

- public-domain content should feel ready
- licensed content should feel premium but blocked until rights are solved

## My Profile

### Required content

- avatar
- name
- email
- phone
- role
- password / security
- delete account warning

### Behavior

- separate profile details from security and destructive settings

## Get More Products

### Required content

- optional future marketplace
- ProContent-equivalent but original
- motion backgrounds / templates
- sermon graphics packs
- church media packs

### Behavior

- not core MVP
- visually richer than the rest of the app, but still aligned
- should not overshadow the main operational product
